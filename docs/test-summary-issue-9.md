## 🔧 Issue #9 での方針

### シンプルで保守性の高いテスト構成

Issue #9 の目的は「認証機能が壊れていないことを保証する」ことです。<br>
以下の3つのテストファイルで十分に達成できています。<br>

1. `test/models/user_test.rb` - User モデルのバリデーションとセキュリティ<br>
2. `test/integration/user_registration_test.rb` - ユーザー登録フロー<br>
3. `test/integration/user_login_test.rb` - ログイン・ログアウトフロー<br>

### 不要なテストファイルは作成しない

- ヘルパーメソッド（`current_user`, `logged_in?`）は統合テストで間接的にテスト済み<br>
- `require_login` メソッドは Issue #10 以降でテスト予定<br>

過剰品質を避け、必要十分なテストに留めることで、保守性の高いテストスイートになっています。<br>
